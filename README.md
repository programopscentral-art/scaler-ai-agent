> AI-powered pre-call briefs and personalised PDFs for Scaler BDAs — generated by Gemini 2.5 Flash, delivered on WhatsApp, gated behind a human approval step.

---

## What It Does

A full-stack sales enablement tool with two core interventions:

| Intervention | Who receives it | When | Gate |
|---|---|---|---|
| **Pre-call WhatsApp brief** | BDA | Before the call | None — sent immediately |
| **Personalised PDF** | Lead (via BDA) | After the call | BDA must approve before send |

The brief gives the BDA: persona analysis, resonant angles, likely objections, and a suggested opening line — all in under 30 seconds.

The PDF addresses each of the lead's specific questions from the call, with Scaler's published outcomes mapped to their profile.

---

## Demo Flow

```
Evaluator enters phones
        ↓
  Lead profile
  (preset or custom)
        ↓
  [1] Generate & Send Nudge  →  BDA receives WhatsApp brief before call
        ↓
  Paste transcript / upload audio
        ↓
  [2] Generate PDF  →  Gemini extracts questions, builds PDF, uploads to Supabase
        ↓
  Approval gate  →  Evaluator reviews, edits cover message, then approves
        ↓
  Lead receives WhatsApp with PDF link
```

## Full Setup

### Prerequisites

- Python 3.10+
- Node.js 18+
- [Google AI Studio](https://aistudio.google.com/) API key
- [Twilio](https://console.twilio.com/) account (free trial works)
- [Supabase](https://supabase.com/) project (free tier works)

### Step 1 — Supabase database

Open your Supabase project → SQL Editor → run the full contents of [`supabase_schema.sql`](supabase_schema.sql).

This creates:
- `sessions`, `leads`, `generated_content` tables (RLS disabled for MVP)
- `pdfs` public storage bucket with anon insert/select/update policies

### Step 2 — Backend

```bash
cd backend
python -m venv .venv
# Windows:
.venv\Scripts\activate
# macOS/Linux:
source .venv/bin/activate

pip install -r requirements.txt
cp .env.example .env
```

Fill in `.env`:

| Variable | Where to find it |
|---|---|
| `GEMINI_API_KEY` | [Google AI Studio](https://aistudio.google.com/) → Get API key |
| `GEMINI_MODEL` | Leave as `gemini-2.5-flash` |
| `TWILIO_ACCOUNT_SID` | Twilio Console → Account Info |
| `TWILIO_AUTH_TOKEN` | Twilio Console → Account Info |
| `TWILIO_WHATSAPP_FROM` | `whatsapp:+14155238886` (sandbox number) |
| `TWILIO_SANDBOX_JOIN_CODE` | Twilio Console → Messaging → Try it out → WhatsApp |
| `SUPABASE_URL` | Supabase → Project Settings → API → Project URL |
| `SUPABASE_KEY` | Supabase → Project Settings → API → `anon` `public` key |
| `BASE_URL` | `http://localhost:8000` for local dev |

```bash
uvicorn main:app --reload --port 8000
```

### Step 3 — Frontend

```bash
cd frontend
npm install
npm run dev    # http://localhost:5173 — proxies /api → :8000
```

For production: `npm run build` writes to `frontend/build/`, served automatically by FastAPI via `StaticFiles`.

### Step 4 — WhatsApp Sandbox opt-in

Both the **BDA number** and the **evaluator number** must opt in to the Twilio sandbox before they can receive messages.

Send the following WhatsApp message to **+1 415 523 8886**:

```
join <your-sandbox-join-code>
```

The sandbox join code is shown in the app's onboarding screen and also in your Twilio console.

| `POST` | `/api/approve` | Approve / edit / skip lead-facing PDF send |

## AI Guardrails (enforced in every prompt)

1. **No fabricated statistics** — Use "based on Scaler's published outcomes", never invented percentages.
2. **No fabricated salaries** — Use reasoning ("a 40–60% jump is typical for this profile"), not invented figures.
3. **Acknowledge uncertainty** — "based on publicly available information" when unsure.
4. **Use the lead's exact words** — Every PDF section paraphrases or quotes what they actually said.
5. **Categorise every question** — `roi/cost`, `curriculum_depth`, `career_outcome`, `program_fit`, `personal_doubt`.

---

## Known Limitations

- **Short transcripts** — Under ~100 words, Gemini extracts surface-level questions and misses emotional subtext. Fix: second Gemini pass targeting underlying fears.
- **Twilio 1600-char limit** — Handled via auto-chunking at paragraph boundaries (`whatsapp_service.py`). Prompt instructs Gemini to stay under 1400 chars.
- **PDF generation latency** — reportlab + two Gemini calls takes 8–12 s per lead. At scale, move to a background job queue (Celery + Redis).
- **Sandbox only** — Twilio WhatsApp Sandbox requires each recipient to send a join code. Production use requires a Twilio approved WhatsApp Business sender.
